git -- version 
--> verifica version de git instalada

git config --global user.name "dbriganty" 
--> asignacion de nombre de usuario 

git config --global user.mail "danielbriganty@gmail.com" 
--> asignacion de email usuario 

git ini 
--> crea repo local debe hacerse en carpeta vacia

git clone direccion_del_repo
-->clonar en carpeta local un proyecto

git add nombre archivo.txt 
--> sube archivo a area de preparacion 

git add .
-->  con el punto se agregan todos los cambios al área de preparación

git commit -m "mensaje del comit" 
--> almacena en el repo los cambios realizados a los archivos presentes en el area de preparación 

git log 
--> Muestra los commits realizados 

git diff commit_id commit_id
--> ver diferencias entre dos commits

git checkout hash_del_commit_previo
--> retorna a la version seleccionada

git tag -a nombre-etiqueta -m "prueba de marcado de etiqueta"   
--> marcar con etiquetas adicionales un commit (aparecen junto a la rama en el promt) 


git branch rama_alternativa
--> crea  una rama

git checkout rama_alternativa
--> te posiciona sobre la rama seleccionada

git checkout -b nombre_rama
--> cambia a rama "nombre_rama" (de no estar creada... la crea.. y se posiciona en ella.)

git merge rama_alternativa
--> se debe realizar desde master o desde la rama a la cual queremos traer los cambios realizadosen rama_alternativa

Al crear desde github un repositorio.. se debe alimentar con los códigos fuente de distintas maneras

creando un repositorio desde la la linea de comando 
…or create a new repository on the command line
echo "# test_repos" >> README.md
git init
git add README.md
git commit -m "first commit"
git remote add origin https://github.com/dbriganty/test_repos.git
git push -u origin master


realizar push desde un repo local ya existente
…or push an existing repository from the command line
git remote add origin https://github.com/dbriganty/test_repos.git
git push -u origin master
